# ポートフォリオ更新・管理ガイド

このポートフォリオは、**「ソースコード（React）」**をビルドして**「公開用データ（HTML/JS/CSS）」**を生成する仕組みになっています。

---

## 1. コンテンツの追加方法

`public/content/` フォルダ内の `.md` ファイルを編集します。

### 基本の書き方
- **見出し（大）**: `# タイトル`（サイト内ではページタイトルになります）
- **見出し（中）**: `## セクション名`（青い左線が入るモダンな見出しになります）
- **見出し（小）**: `### 項目名`（下線付きの強調表示になります）
- **箇条書き**: `- 項目`
- **リンク**: `[表示テキスト](URL)`

### 画像の追加
1. 画像ファイルを `public/assets/` に保存します。
2. Markdown内に次のように記述します：
   `![説明文](/assets/ファイル名.png)`
   ※自動的に中央寄せされ、角丸・影付きのモダンなスタイルで表示されます。

---

## 2. 【重要】本番環境への反映手順（方法B）

ソースコードのリポジトリと、本番公開用のリポジトリが分かれている場合、以下の手順で手動ビルドして反映させます。

### ステップ1：ビルド（変換）を実行する
ターミナル（PowerShellなど）で以下のコマンドを実行します。
```powershell
npm run build
```
実行が終わると、プロジェクト内に **`dist/`** というフォルダが作成されます。これが「本番用データ」です。

### ステップ2：本番環境へコピーする
1. 作成された **`dist/` フォルダの中身すべて**（`assets/` フォルダ、`content/` フォルダ、`index.html` など）をコピーします。
2. **本番公開用（別で管理されている）リポジトリのルートディレクトリ**に貼り付けます。
   - ※以前の古いファイル（`index.html` など）は上書きしてください。

### ステップ3：本番リポジトリでプッシュする
本番リポジトリ側で通常通り `git add`, `git commit`, `git push` を行えば、Webサイトが更新されます。

---

## 3. 将来的な自動化について（方法A）

将来的に「ソースをプッシュしたら自動で本番リポジトリまで更新したい」となった場合は、GitHubの **Personal Access Token (PAT)** を取得し、GitHub Actions の設定ファイルを書き換えることで実現可能です。

その準備ができた際は、再度指示をいただければ設定ファイルを修正します。

- **スタイリング**: Tailwind CSS v4
- **Markdown表示**: react-markdown + Tailwind Typography
- **アイコン**: lucide-react
